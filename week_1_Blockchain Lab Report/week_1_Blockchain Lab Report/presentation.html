CoinGecko API

무료 암호화폐 가격 API 이해하기

API 요청 URL 분석 API

HTTP GET https://api.coingecko.com/api/v3/simple/price ?ids=bitcoin,ethereum // 조회할 코인 &vs_currencies=usd,krw // 달러 + 원화 &include_24hr_change=true // 24시간 변동률 &include_market_cap=true // 시가총액

응답 예시 (JSON)

JSON { "bitcoin": { "usd": 66645, "krw": 98302614, "usd_24h_change": 0.19, // +0.19% 상승 "usd_market_cap": 1334108163262 } }

핵심 코드 분석

crypto-prices.html의 JavaScript 로직

tryFetch() — API 응답 검증 Core

JavaScript async function tryFetch(url) { try { const response = await fetch(url); // 1. HTTP 요청 if (!response.ok) return null; // 2. 상태코드 확인 (200?) const data = await response.json(); // 3. JSON 파싱 if (data && data.bitcoin && data.bitcoin.usd) return data; // 4. 구조 검증 return null; } catch (e) { return null; // 네트워크 에러 → null } }

포인트: 2단계 검증 — HTTP 상태코드 확인 + 응답 데이터 구조 확인. 실패하면 null 반환하여 다음 프록시로 넘김

fetchPrices() — 폴백 전략 Core

JavaScript // CORS 프록시 목록 const CORS_PROXIES = [ (url) => `allorigins.win/...`, (url) => `corsproxy.io/...`, (url) => `codetabs.com/...`, ]; async function fetchPrices() { // 1단계: 직접 호출 let data = await tryFetch(API_URL); // 2단계: 실패 시 프록시 순회 if (!data) { for (const proxy of CORS_PROXIES) { data = await tryFetch(proxy(API_URL)); if (data) break; } } if (data) updateTable(data); }

왜 프록시가 필요한가?

  • 브라우저는 CORS 정책으로 다른 도메인 API 호출을 차단
  • file://에서 열면 거의 100% 차단됨
  • 프록시 서버가 대신 호출 → 브라우저에 전달

참고: Next.js를 쓰면 서버에서 직접 호출하므로 CORS 문제 자체가 없음

updateTable() — 테이블 렌더링 Core

JavaScript function updateTable(data) { const coins = [ { id: "bitcoin", name: "Bitcoin (BTC)" }, { id: "ethereum", name: "Ethereum (ETH)" }, ]; coins.forEach((coin) => { const info = data[coin.id]; // data["bitcoin"] const change = info.usd_24h_change; // 24시간 변동률 const changeColor = change >= 0 ? "text-green-600" // 상승 → 초록 : "text-red-600"; // 하락 → 빨강 tr.innerHTML = `...${coin.name}...${info.usd}...`; tbody.appendChild(tr); // DOM에 추가 }); }

포인트: API 응답의 usd_24h_change 값으로 상승(+) / 하락(-) 색상을 동적으로 결정

resetCountdown() — 30초 자동 갱신 Core

JavaScript const REFRESH_INTERVAL = 30; function resetCountdown() { countdown = REFRESH_INTERVAL; countdownTimer = setInterval(() => { countdown--; // 1초마다 감소 // 프로그레스 바 너비 계산 const percent = (countdown / REFRESH_INTERVAL) * 100; progressBar.style.width = percent + "%"; if (countdown <= 0) { clearInterval(countdownTimer); fetchPrices(); // 다시 호출! } }, 1000); // 1000ms = 1초 }

동작 원리

  • setInterval로 1초마다 콜백 실행
  • 카운트다운 숫자 감소 + 바 너비 줄임
  • 0초 도달 → fetchPrices() 재호출
  • 이것이 30초 자동 갱신 사이클

실전 이슈

교실에서 겪을 수 있는 문제와 해결법

Rate Limit 문제 Issue

왜 교실에서 안 되나?

교실 네트워크 학생A ──┐ 학생B ──┤ 학생C ──┼── 교실 Wi-Fi ── 공인 IP 1개 ── CoinGecko ... │ 학생30 ─┘ CoinGecko 입장: → "하나의 IP에서 요청 폭주!" → HTTP 429 Too Many Requests

30명 x 2회/분 = 분당 60회
무료 한도: 분당 10~30회

해결 방법

방법난이도
갱신 주기 30초 → 120초쉬움
CoinGecko Demo Key 발급 (무료)쉬움
모바일 핫스팟 사용 (IP 분리)쉬움
서버 프록시 + 캐시중간

Tip: 에러가 뜨면 1분 정도 기다리면 자동 복구됩니다

CORS란? Concept

CORS (Cross-Origin Resource Sharing)

  • 브라우저의 보안 정책
  • 다른 도메인의 API 호출을 기본적으로 차단
  • 악성 사이트가 몰래 API를 호출하는 걸 방지
차단 예시 내 페이지: file:///crypto-prices.html ↓ fetch() API서버: https://api.coingecko.com ↓ 브라우저: "출처가 다르니까 차단!" → CORS Error

이 프로젝트의 해결법

방법 1: CORS 프록시 (HTML 버전)

중간 서버가 대신 요청 → 서버↔서버는 CORS 제한 없음

방법 2: 서버 사이드 호출 (Next.js 버전)

Route Handler (/api/prices)에서 서버가 직접 호출 → CORS 문제 자체가 없음

결론: CORS는 브라우저에서만 발생. 서버에서 호출하면 해결!
